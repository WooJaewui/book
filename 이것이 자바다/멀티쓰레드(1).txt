
 멀티쓰레드(1)


 # 멀티쓰레드 관련 용어.

프로세스 : 실행 중인 하나의 프로그램.
멀티태스킹 : 두 가지 이상의 작업을 동시에 처리하는 것.
다중프로세스 : 하나의 어플리케이션(프로그램)을 실행했을 때 여러 프로세스가 생성될 수 있다.
싱글쓰레드 : 하나의 프로세스에 하나의 쓰레드만 있는 것을 의미한다.
멀티쓰레드 : 하나의 프로세스에서 멀티태스킹을 할 수 있는 기능이다. ( 두 개 이상의 쓰레드 )

멀티쓰레드와 다중프로세스 차이.
다중프로세스 : 프로세스가 각각 독립적이다. ( 오류가 발생해도 다른 프로세스는 작동 )
멀티쓰레드 : 오류가 발생하면 프로세스 자체가 종료 될 수 있다. ( 모든 쓰레드가 종료 )


 # main 쓰레드.

설명.
모든 Java application은 main( )를 실행해서 main Thread가 생성되면서 시작된다.

특징.
모든 쓰레드가 종료되어야 프로세스가 종료된다.


 # Thread 생성.

2가지 방식.
1. Thread 클래스 상속받기.
2. Runnable 인터페이스 구현하기.
3. 익명객체로 

설명.
두 방식 모두 run( ) 안에 실행할 문장을 Overriding해야된다.
start( )를 호출하면 start( )가 run( )을 호출하여 실행한다. 

Thread 클래스 방식으로 사용.
OverClass o = new OverClass( );
o.start( );

Runnable 인터페이스 방식으로 사용.
ImplClass i = new ImplClass( );
Thread t = new Thread( i );
t.start( );

주의사항.
Runnable 인터페이스를 구현한 클래스는 start( )가 없기 때문에
Thread Class의 생성자에 매개변수로 대입하여 객체 생성 후 Thread.start( )한다.


 # 쓰레드의 이름.

설명.
디버깅할 때 쓰레드 이름이 따로 정해져 있으면 편리하다. ( 큰 역할이 있는 것은 아님 )
main method의 쓰레드 이름은 "main"이고, 이름을 정하지 않으면 "Thread-n"이 된다.

사용.
Thread thread = new Thread( );
thread.setName( "쓰레드 이름" );



 # 쓰레드 우선순위 관련 용어.

용어설명.
동시성 Concurrency : 멀티 쓰레드가 번갈아가며 실행하는 성질. ( Java thread )
병렬성 Parallelism : 멀티 코어에서 개별 쓰레드가 동시에 실행하는 성질.
우선순위 : 동시성 상황에서 어떤 쓰레드를 먼저 실행할지 결정하는 것.



 # 자바의 우선순위.

자바 스케쥴링 2가지 방법.
1. 우선순위. ( Priority )
2. 순환할당. ( Round-Robin )

순환할당 방식은 시간 할당량 ( Time Slice )을 정해서 실행하는 방식으로 코드로 제어 할 수 있는 방법이 없다.

우선순위설정.
thread.setPriority( 0~10 ) ;

주의사항.
Priority를 따로 설정하지 않으면 5로 설정된다.
10으로 설정하면 가장 우선순위가 높게 측정된다.
순환할당과 우선순위를 같이 사용함으로 우선순위를 10으로해도 무조건 빨리 되는 것은 아니다.
