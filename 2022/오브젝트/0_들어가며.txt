 # 패러다임.

패러다임.
- 과거 패러다임은 패턴, 모델 등을 의미한다.
- 최근에는 어떠한 지금 시대에 사용하는 이론이나 방법, 문제의식 등을 의미하기도 한다.

패러다임의 변화.
- 패러다임의 변화는 이전에 알고 있던 지식이나 이론이 새로운 이론으로 바뀌는 것을 의미한다.
- 개발자의 패러다임의 대표적인 에로는 절차형 -> 객체형 프로그래밍이 있다.



 # 프로그래밍 패러다임.

프로그래밍 패러다임.
- 특정 시대의 어느 성숙한 개발자 공동체에 의해 수용된 프로그래밍 방법과 문제 해결 방법, 프로그래밍 스타일을 의미한다.
- 쿤의 패러다임은 두 가지의 패러다임은 공존할 수 없다고 말하지만, 프로그래밍 패러다임은 두 개의 패러다임이 공존할 수 있다.
