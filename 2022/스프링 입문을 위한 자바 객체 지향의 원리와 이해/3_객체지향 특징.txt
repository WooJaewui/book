 # 객체지향 키워드.

스태틱 영역.
- 패키지, 클래스들은 처음 사용될 때 스태틱 영역에 로드된다.

static 블럭.
- 클래스의 첫번째 객체를 생성될 때 한번만 실행되는 블럭.
- 일반적으로는 객체가 생성될 때 실행되고, static 멤버(변수 또는 메서드)를 사용하면 static 블럭도 같이 사용된다.
- @BeforClass 어노테이션 찾아보기.

인스턴스 블럭.
- 객체가 생성될 때마다 실행되는 블럭.

final.
- 제어자 중 하나이다.
- 클래스, 변수, 메서드 앞에 사용할 수 있다.
- 클래스 앞에 붙으면 상속할 수 없다.
- 변수 앞에 붙으면 값을 수정할 수 없다. (상수)
- 메소드 앞에 붙으면 override할 수 없다.

final 변수.
- static final 변수는 선언할 때 또는 static 블럭을 통해 초기화할 수 있다.
- 인스턴스 final 변수는 선언할 때 또는 생성자 또는 인스턴스 블럭을 통해 초기화할 수 있다.

instanceof 연산자.
- 객체가 특정 클래스의 인스턴스인지 확인할 수 있는 연산자.
- true, flase로 값을 반환한다.
- 변수 instanceof 클래스(타입) == true/false;

변수 특징.
- 지역 변수와 속성의 이름이- 같은 경우 지역 변수가 우선ㅅ한다.
- 객체 변수와 이름이 같은 지역 변수가 있는 경우 객체 변수를 사용하려면 this 접두사로 사용한다.
- 정적 변수와 이름이 같ㅇ느 지역 변수가 있는 경우 정적 변수를 사용하려면 클래스명을 접두사로 사용한다.

super.
- 바로 위 상위 클래스의 인스턴스를 지칭하는 키워드.

JVM 메서드.
- 스태틱 영역에 클래스에 메서드를 하나만 남긴다.
- 모든 인스턴스를 생성할 때마다 중복되는 메서드를 생성하면 메모리 공간 낭비가 심하기 때문에.
