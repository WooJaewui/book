
 JPA 개념.



 # ORM.

ORM의 필요성.
자바의 객체와 데이터베이스의 테이블이 정확하게 일치하지 않는 문제를 해결해준다.

ORM Object Relation Mapping.
정확하게 일치하지 않는 자바 객체와 테이블 사이를 매핑해준다.
DBMS가 변경되면 ORM 환경설정에 변경되었다는 것만 설정해주면 자동으로 SQL을 생성한다.



 # JPA의 특징.

JPA Java Persistence API.
모든 ORM 구현체( ORM 프레임워크 )들의 공통 인터페이스를 제공한다.
JPA API의 인터페이스들을 이용하면 실질적인 DB 연동 처리는 드라이버 클래스들이 담당한다.
DBMS가 변경되는 상황에서도 드라이버만 변경하면 JPA API를 이용하는 애플리케이션은 수정하지 않는다.
