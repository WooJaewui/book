 # URL 문법.

URL과 URI.
URL : 리소스의 위치.
URI : 리소스 식별자.

URL.
스킴 : 웹 클라이언트가 리소스에 어떻게 접근하는지 알려준다. ( http, https )
호스트명 : 서버의 위치로 리소스가 어디에 호스팅 되어 있는지 알려준다. ( www.naver.com )
리소스 : 리로스 경로를 의미하고 서버에 존재하는 리소스들 중에서 요청받은 리소스를 알려준다. 
'스킴://호스트명//경로' 구조로 이루어져 있다.

스킴.
주어진 리소스에 어떻게 접근하는지 알려주는 중요한 정보다.

호스트와 포트.
호스트 : 리소스를 호스팅하고 있는 장비를 알려준다.
포트 : 호스팅하고 있는 장비 내에서 리소스에 접근할 수 있는 서버를 알려준다. ( HTTP 기본포트 80 )

사용자 이름과 비밀번호.
데이터 접근을 허용하기 전에 사용자 이름과 비밀번호를 요구한다.
아이디 비밀번호가 입력되지 않은 경우 자동으로 기본 사용자 아이디 비밀번호가 입력된다.

경로.
리소스가 서버의 어디에 있는지 알려준다.

파라미터.
리소스를 찾기 위한 더 많은 정보가 필요할 때 사용한다.

질의 문자열 query String.
요청받을 리소스 형식의 범위를 좁히기 위해서 질문이나 질의를 받을 수 있다.

프래그먼트.
리소스 안에 특정한 부분을 가르킬 수 있다.
'URL#프래그먼트' 를 통해 사용할 수 있다.



 # 단축 URL.

절대 URL.
일반적으로 사용하는 URL방식.

상대 URL.
base URL을 기준으로 '.' , '..'을 활용해서 URL을 표현하는 방식.

URL 확장.
호스트명 확장 : 웹 브라우저에 호스트명만 작성해도 자동으로 URL을 확장해준다.
히스토리 확장 : 사용자가 방문했던 URL의 기록을 저장해 완성된 글자를 표시해준다.



 # 안전하지 않은 문자.

이스케이프 문자.
안전하지 않은 문자를 안전한 문자로 인코딩할 수 있게 도와주는 문자.
