 # 내용 협상 기법.

내용 협상 content-negotiation.
하나의 URL이 여러 가지 리소스 중 적합한 것에 대응할 수 있도록 하는 방법이다.

내용 협상 기법 종류.
클라이언트 주도, 서버 주도, 투명.



 # 클라이언트 주도 협상.

클라이언트 주도 협상.
클라이언트가 보고 싶은 것을 선택하게 하는 방식.
서버 입장에서 가장 구현하기 쉽다.
단점은 각 페이지에 두 번의 요청이 필요하다는 것이다.



 # 서버 주도 협상.

서버 주도 협상.
서버가 어떤 페이지를 돌려줄 것인지 결정하는 방식이다.
클라이언트에서 선호하는지에 대한 충분한 정보를 서버에게 전달해야 된다.

내용 협상 헤더.
Accept, Accept-Language, Accept-Charset, Accept-Encoding 등을 통해 정보를 제공.

아파치의 내용 협상.
웹 사이트 컨텐츠 제공자에게 내용 협상을 결정하게 하는 방식이다.
아파치 서버에 설정을 통해 헤더 내용을 보고 판단하여 리소스를 제공하는 방식이다.



 # 투명 협상.

투명 협상.
클라이언트 입장에서 협상하는 중개자 프락시를 둠으로써 메시지 교환을 최소화하는 방식이다.

Vary 헤더.
서버가 문서를 선택할 때 고려한 클라이언트 요청 헤더 모두를 나열한다.
Vary 헤더의 목록에 있는 것들이 캐쉬에 헤더에 존재하지 않으면 다시 서버에 요청해야 된다.



 # 트랜스 코딩.

트랜스 코딩.
서버가 클라이언트의 요구에 맞는 문서를 갖고 있지 않을 때 기존 문서를 변경해서 응답하는 경우.

포맷 변환.
데이터를 클라이언트가 볼 수 있도록 다른 포맷으로 변환하는 것.

정보 합성 information synthesis.
문서에서 정보의 요점을 추출하는 방법.

컨텐츠 주입.
문서에서 정보를 추가하는 방법.
