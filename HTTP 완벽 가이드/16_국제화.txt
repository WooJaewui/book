 # 문자 집합과 HTTP.

서버.
클라이언트에게 Content-Type charset과 Content-Language 헤더를 통해 문서의 언어를 알려준다.

클라이언트.
서버에게 Accept-Charset과 Accept-Language 헤더를 통해 이해할 수 있는 언어를 알려준다.

Charset은 글자를 비트로 변환하는 인코딩이다.
엔티티 컨텐츠 비트들을 어떻게 특정 문자 체계로 바꾸는지 알려준다.
Charset 태그는 등록된 MIME 문자집합에 표준화되어 있고, IANA가 관리한다.

문자집합과 인코딩은 어떻게 동작하는가.
문서를 이루는 비트들은 문자집합으로 디코딩되어 문자 코드로 변환된다.
글자의 모양은 그래픽 디스플레이 소프트웨어( 브라우저, 운영체제, 글꼴 )가 결정한다.

charset 오류.
잘못된 charset을 사용하면 깨진 글자를 보여주게 된다.



 # 다중언어 문자 인코딩에 대한 지침.

문자집합 용어.
문자 : 알파벳 글자, 숫자, 구두점, 기호 등 글쓰기의 최소 단위.
글리프 glyph : 하나의 글자를 표현하기 위한 시각적 형태.
코딩된 문자 coded character : 각 글자에 할당된 유일한 숫자.
코드 공간 coding space : 문자 코드 값으로 사용하려고 계획해 둔 정수의 범위.
코드 너비 code width : 각 문자 코드의 비트 개수.
사용 가능 문자집합 : 글자들에 대한 특정한 작업 집합.

US-ASCII.
정보교환을 위한 미국 표준코드로 표준화된 가장 유명한 코딩된 문자집합이다.
0~127 가지 값만 사용해서 표현하는데 7비트 공간만 필요한다.

UCS Universal Character Set.
전 세계의 모든 글자를 하나의 코딩된 문자집합으로 통합하려 노력하는 표준이다.
유니코드는 UCS 표준을 따르는 상업적인 컨소시엄이다.

고정폭
문자를 고정된 길이 비트로 표현. 
빠르게 처리 될 수 있지만 공간 낭비 우려.

가변폭 - 비모달.
문자 코드 번호에 다른 길이의 비트를 사용한다.
자주 사용하는 글자의 비트 길이를 줄일 수 있다.

UTF-8 UCS Transformation Format.
인기 있는 UCS를 위해 설계된 문자 인코딩 구조이다.
비모달 가변길이 인코딩을 사용한다.



 # 언어 태그와 HTTP.

Content-Language 헤더.
엔티티가 어떤 언어 사용자를 대상으로 하고 있는지 서술한다.
en , de , ko 등 다른 언어에 대한 언어 태그가 존재한다.

언어 태그의 종류.
일반적인 언어의 종류.
특정 국가의 언어.
그 밖에 방언, 지방어, 비표준 언어 등.

IANA 언어 태그 등록.
IANA는 RFC 3066의 규칙에 따라 표준 언어 태그의 목록을 관리한다.



 # 국제화된 URI.

제한된 URI.
URI는 ASCII 문자들의 집합으로만 사용할 수 있도록 설계되었다.

URI에서 사용될 수 있는 문자들.
예약어 : [ ; , / , ? , : , ... 등 ]
예약되지 않음 : [ A-Z , a-z , 0-9 , - , ... 등 ]
이스케이프 : [ % , <HEX> ]

이스케이핑과 역이스케이핑.
URI에 %를 사용해서 띄어쓰기를 표현할 수 있다.
%를 직접 입력하려면 %와 매핑된 문자번호인 25를 입력하면 %가 된다.



 # 기타 고려사항.

헤더와 명세에 맞지 않는 데이터.
HTTP 헤더는 반드시 US-ASCII 문자집합의 글자들로만 이루어져야 한다.

날짜.
HTTP 명세는 올바른 GMT 날짜 형식을 정의하지만 웹 서버와 클라이언트가 규칙을 따르지는 않는다.
