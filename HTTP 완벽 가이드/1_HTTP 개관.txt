 # HTTP 개관.

웹 클라이언트와 서버.
웹 서버 : 인터넷의 데이터를 저장하고, HTTP 클라이언트가 요청한 데이터를 제공한다.
클라이언트 : 서버에게 HTTP 요청을 보낸다.

리소스.
웹 리소스는 웹 서버가 관리하고 제공한다.
정적 파일 + 프로그램 -> 모든 종류의 컨텐츠 소스.

미디어 타입 - MIME Multipurpose Internet Mail Extensions.
MIME은 최초에 각기 다른 전자메일 시스템 사이에 문제점을 해결하기 위해 개발되었다.
현재는 HTTP에서 데이터 포맷을 나타낼때 사용한다.
주 타입 / 부 타입으로 이루어진 문자열 라벨이다.
예 : text/html , text/plain , image/jpeg , ... 

URI Uniform Resource Identifier.
클라이언트가 요청을 할 때 사용하는 리소스 식별자이다.
URL과 URN 두가지 종류가 있는데, URL을 거의 많이 사용한다.
URL과 URI는 거의 같은 의미로 사용된다.
스킴 scheme : http:// , https://와 같은 프로토콜.
인터넷 주소 : www.naver.com.
리소스 : /abc/gg.jpg. 

트랜잭션.
요청 명령과 응답 명령이 하나의 트랜잭션으로 구성되어 있다.

메소드.
GET : 서버에서 클라이언트로 지정된 리소스를 보내라.
PUT : 클라이언트에서 서버로 보낸 데이터를 지정한 이름의 리소스로 저장해라.
DELETE : 리소스를 서버에서 삭제해라.
POST : 클라이언트 데이터를 서버 게이트웨이 애플리케이션으로 보내라.
HEAD : 지정한 리소스에 대한 응답에서, HTTP 헤더 부분만 보내라.

상태코드.
응답 메시지를 보낼 때 함께 보내는 응답 결과 정보이다.
요청이 성공했는지 추가 조치가 필요한지 등을 알려준다.

메시지.
요청 메시지와 응답 메시지가 있다.
시작줄 : 요청 - 무엇을 해야 하는지 보여줌, 응답 - 무슨 일이 일어났는지 보여줌.
헤더 : ' : '으로 구분되어 하나의 이름과 하나의 값으로 구성된다.
본문 : 요청 - 웹 서버로 데이터를 보냄 , 응답 - 클라이언트로 데이터를 반환.

TCP / IP.
오류 없는 데이터 전송.
순서에 맞는 전달 ( 보낸 순서대로 도착 )
조각나지 않는 데이터 스트림 ( 어떤 크기로든 보낼 수 있다 )

TCP / IP 실행 순서.
웹 브라우저는 URL에서 호스트 명을 알아낸다.
웹 브라우저는 호스트 명을 IP로 변환한다.
웹 브라우저는 URL에서 포트번호를 추출한다.
웹 브라우저는 웹 서버와 TCP 커넥션을 맺는다.
웹 브라우저는 서버에 HTTP 요청을 보낸다.
서버는 웹 브라우저에 HTTP 응답을 보낸다.
커넥션을 끊고, 웹 브라우저는 문서를 보여준다.

웹의 구성요소.
프록시 : 클라이언트와 서버 사이에 위치한 HTTP 중개자.
캐시 : 최근 방문했던 문서들을 저장하는 프록시 서버.
게이트웨이 : 다른 서버들의 중개자로 동작하는 서버.
터널 : 두 커넥션 사이의 데이터를 열어보지 않고 그대로 전달해주는 HTTP 애플리케이션.
에이전트 : HTTP 요청을 만들어주는 클라이언트 프로그램.
